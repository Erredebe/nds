Integracion host Shadow vs light Decision tree

Host integration lab

Este caso resume cuando conviene mantener aislamiento y cuando abrir el DOM al host. La idea no es elegir un modo por gusto, sino por restricciones reales de integracion.

Shadow por defecto

Ideal cuando el host comparte CSS entre muchos equipos y no puedes garantizar estabilidad.

Mixed / light selectivo

Util cuando solo uno o dos componentes necesitan heredar estilos del host o exponer su DOM final.

Empieza por shadow

Es el default recomendado cuando todavia no conoces del todo el host y quieres aislamiento estable.

Abre light con motivo

Usalo cuando el host necesita hooks de estilo, tracking o inspeccion del DOM final.

Mixed como compromiso

Evita decisiones globales y abre solo las piezas que de verdad necesitan convivir con el host.

Cuadro de decision

Una regla simple: empieza en `shadow`; abre `light` solo cuando el host lo justifique de verdad.

Si dudas, la opcion mas segura sigue siendo `shadow`. Cambia a `light` solo cuando puedas nombrar la necesidad de integracion con una frase concreta.
  • Usa `shadow` si el host tiene CSS global agresivo o varios equipos tocan la misma pagina.
  • Usa `light` si necesitas style hooks externos o inspeccion del DOM final por parte del host.
  • Usa `mixed` si solo ciertos componentes requieren apertura y el resto debe seguir aislado.

Que demuestra este caso

No basta con decir que existen dos modos. La demo debe explicar por que y cuando cambia la decision.

Aislamiento Protege el componente cuando el host no es fiable.
Integracion Permite heredar estilos o exponer nodos cuando el host lo necesita.
Gradualidad Mixed evita decisiones globales cuando solo hay excepciones puntuales.

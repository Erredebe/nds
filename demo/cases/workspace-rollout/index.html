Workspace rollout

Caso completo para evaluar un flujo de integracion: captura de contexto, simulacion de despliegue, timeout, aprobacion pendiente y recuperacion visible sin romper el draft del usuario.

Que valida este caso

La pregunta no es si los componentes se ven bien, sino si el flujo se sostiene cuando aparecen fricciones.

  • Validacion previa al submit.
  • Draft persistido entre recargas.
  • Estado `loading`, `success` y error recuperable.
  • Payload visible preparado por el host.
  • Decision tree para `shadow`, `light` y `mixed`.

Prueba rapida recomendada

Haz estas tres acciones para percibir la fiabilidad sin leer codigo.

  1. Ejecuta el happy path y revisa el payload.
  2. Fuerza un timeout y comprueba el playbook de recuperacion.
  3. Recarga la pagina para verificar la restauracion del draft.
Consejo: prueba tambien solo con teclado. El caso gana credibilidad cuando puedes recorrerlo, dispararlo y entender el resultado sin depender del raton.

DOM modes

La comparativa se resuelve por pagina para evitar redefinir custom elements dentro del mismo documento. Cada iframe tiene su propio registro.

Comparativa visual

Izquierda: configuracion default en shadow DOM. Derecha: pagina dedicada con button: { dom: 'light' }.

Shadow por defecto

Todos los componentes mantienen el modo configurado en su definicion.

Mixed / light

Solo nds-button se registra en light DOM; el resto se mantiene en shadow.

Configuracion usada

La demo principal no redefine tags. La pagina mixed.html aplica el override al cargar.

Config 
defineAllComponents({
  button: { dom: 'light' }
});

setTheme('dark');

Criterio de uso

Usa overrides de DOM mode solo cuando haya una necesidad real de integracion o style hooks del host.

  • shadow sigue siendo el default recomendado para aislamiento.
  • light resulta util cuando el host necesita exponer el DOM final a estilos externos.
  • El registro es idempotente: la primera definicion sigue ganando.
